## 🏭 What is Trier OS?

**Trier OS** is an enterprise-grade, full-stack Plant Operations system built for heavy manufacturing and industrial facilities. It is not maintenance software — it is a **Living OS**.

It combines central operational controls, 3D GIS spatial intelligence, mobile hardware scanning, and a unique built-in **sandbox IDE** — allowing authorized operators to safely modify the system directly from the plant floor, with zero reliance on external cloud infrastructure.

---

## 🚀 Core Engines

| Engine | Description |
|---|---|
| 🛠️ **Live Studio Sandbox** | Embedded Monaco-based IDE. Write, sandbox, and hot-reload source code from inside the running production app. |
| 🌌 **Parallel Universe Engine** | Deterministic simulation engine. Replays historical plant event logs against sandboxed code changes — mathematical proof before deployment. |
| 💰 **Frictional Cost Engine** | Static UX analyzer that calculates the exact financial "wrench-time" cost of UI changes before they ship. |
| 🌍 **GIS Spatial Intelligence** | Full 3D spatial intelligence maps (Cesium) to pinpoint hardware assets across corporate campuses. |
| 📱 **Mobile Hardware Scanning** | WebRTC barcode scanning for real-time audit sweeps on iOS, Android, and Zebra rugged devices. |
| 🔒 **EDR-Safe Local Mode** | Runs fully air-gapped using a self-contained `better-sqlite3` instance — built for firewalled OT networks. |
| 🛡️ **Human Airgap Security** | Hard security boundary. All AI assistance is decoupled from the plant network and strictly human-mediated. |

## ⚡ Quick Start

```bash
git clone https://github.com/DougTrier/trier-os.git
cd trier-os
npm install
npm run dev:full
```

App boots at `http://localhost:5173` — no cloud account required.

## 📐 Architecture Philosophy

Trier OS is built on **extreme contextual transparency** — a strict ~10.2% Contextual Density Ratio across 349,000+ lines of code.

Every logic file carries the **Trier OS Architecture Header Pattern**, ensuring any engineer can understand *why* a function exists in the context of physical Plant Floor Operations — not just *what* it does.

> **Contributing?** Any PR that drops contextual coverage below 100% is auto-rejected. See [`CONTRIBUTING.md`](https://github.com/DougTrier/trier-os/blob/main/CONTRIBUTING.md).
